# Dev.to Platform Guide

**Best for:** Technical articles, developer audience, SEO
**Optimal length:** 4-7 min read
**Frequency:** 1-2x/week
**Format:** Long-form markdown articles with code

## What works
- Tutorials with runnable code examples
- Problem-solution format ("How to fix X")
- Comparative posts ("X vs Y: Which is better?")
- Deep dives into specific technologies
- Career advice for developers
- Open source project showcases

## Article structure
```
Title (specific outcome, not generic — "How to X" not "X Explained")
---
Cover image with text overlay
---
Problem statement (hook)
Context (why this matters)
Step-by-step solution (code snippets)
Results/verification
Key takeaways (bullet points)
CTA (follow, comment, check out repo)
```

## Tags
- 3-5 relevant tags per post
- Place after title in frontmatter
- Popular: #python, #javascript, #go, #tutorial, #opensource
- Be specific, not generic

## Best practices
- Open with a specific problem ("I was debugging X for 3 hours...")
- Every code block must be runnable or clearly explained
- One concept per article, deeply
- Add a TL;DR at the top for skimmers
- Cover image with text overlay (1050x600 recommended)
- Engage with comments in first 24 hours
- Cross-post from your blog with canonical URL

## SEO tips
- Title includes primary keyword
- Description in frontmatter (150-160 chars)
- Use headings (H2, H3) for structure
- Alt text on images
- Internal links to related Dev.to posts
